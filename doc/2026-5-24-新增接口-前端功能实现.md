# 2026-5-24 新增接口前端功能实现

## 1. 实现目标

根据 `2026-5-24-新增接口说明.md`，前端已接入管理员资料接口，用于修改当前登录管理员昵称。

本次实现将原后台右上角的“修改密码”入口收拢到侧边栏“管理员信息”页面，并在同一页面提供：

- 修改管理员昵称
- 修改管理员密码

同时，文章评论区已增加管理员评论标识：当评论作者用户名匹配站长用户名时，在昵称旁展示“站长”标签。

## 2. 页面入口

后台侧边栏新增菜单：

| 菜单 | 路由 | 说明 |
| --- | --- | --- |
| 管理员信息 | `/admin/account` | 管理当前管理员昵称和密码 |

兼容旧路由：

| 旧路由 | 行为 |
| --- | --- |
| `/admin/account/password` | 渲染同一个管理员信息页面 |

后台顶部右侧已移除单独的“修改密码”按钮，保留“返回前台”和“退出”。

## 3. 接口接入

### 3.1 修改管理员昵称

```http
PATCH /api/admin/me/profile
```

请求体：

```json
{
  "nickname": "Sam"
}
```

成功后使用接口返回的 `UserView` 更新前端当前用户状态。

### 3.2 修改管理员密码

继续沿用已有接口：

```http
PATCH /api/admin/me/password
```

修改成功后沿用原行为：提示“密码已修改，请重新登录”，然后退出登录。

## 4. 状态更新策略

昵称修改成功后，前端不会额外调用 `/api/auth/me` 刷新用户信息，而是直接信任 `PATCH /api/admin/me/profile` 返回的 `UserView`。

前端同步更新：

- `AuthContext` 中的当前用户状态
- `localStorage` 中的当前用户信息
- 后台顶部“当前用户”展示
- 前台布局中的用户昵称按钮

由于接口说明提到 `/api/auth/me` 可能仍取到 token 中的旧昵称，登录态初始化刷新时会保留本地已更新的同一用户昵称，避免页面刷新后被旧昵称覆盖。

## 5. 前端校验和错误处理

管理员昵称表单校验：

- 必填
- 提交前去除首尾空格
- 长度 1-64

密码表单校验：

- 旧密码必填
- 新密码必填
- 新密码长度 6-64

接口错误继续使用统一的 `showApiError` 展示后端返回信息。

## 6. 评论站长标识

评论区渲染时使用评论作者用户名判断是否为管理员评论。

配置项：

```env
VITE_ADMIN_USERNAME=admin
```

规则：

- 前端读取 `VITE_ADMIN_USERNAME` 作为站长用户名。
- 未配置时默认使用 `admin`。
- 当 `comment.author.username === VITE_ADMIN_USERNAME` 时，在昵称旁展示 `站长` 标签。
- 该功能依赖用户名稳定不变，不依赖昵称；管理员修改昵称后仍可正常标识。

涉及页面：

| 页面 | 行为 |
| --- | --- |
| 文章详情评论区 | 管理员评论昵称旁展示“站长” |

## 7. 验收记录

建议验收步骤：

1. 管理员登录后台。
2. 确认侧边栏出现“管理员信息”。
3. 进入 `/admin/account`，确认昵称输入框回填当前昵称。
4. 修改昵称并保存，确认后台顶部“当前用户”立即更新。
5. 刷新页面，确认昵称仍保持为新值。
6. 访问 `/admin/account/password`，确认仍打开同一个管理员信息页面。
7. 修改密码，确认成功后退出登录。
8. 使用管理员账号发表评论，确认评论昵称旁展示“站长”。
9. 使用游客账号发表评论，确认不会展示“站长”。
