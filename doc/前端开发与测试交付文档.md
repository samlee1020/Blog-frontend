## 2. 本机启动后端

### 2.1 推荐方式：Docker Compose

前置条件：

- Docker 可用。
- `docker compose version` 能正常输出版本。
- 本机端口 `8080`、`3306`、`6379` 未被占用。

启动命令：

```bash
cd /Users/samlee/Desktop/Code/Blog-backend
docker compose up --build -d
```

首次启动会下载 Maven 依赖并构建镜像，耗时可能较长。后续构建命中缓存会明显变快。

检查容器状态：

```bash
docker compose ps
```

期望看到：

- `blog-backend-mysql-1` 为 `Up` 且 `healthy`。
- `blog-backend-redis-1` 为 `Up` 且 `healthy`。
- `blog-backend-app-1` 为 `Up`。

检查启动日志：

```bash
docker compose logs app --tail=120
```

看到 `Tomcat started on port 8080` 即表示后端启动完成。

默认访问地址：

| 服务 | 地址 |
| --- | --- |
| 后端 API | `http://localhost:8080` |
| MySQL | `localhost:3306` |
| Redis | `localhost:6379` |
| 上传文件 | `http://localhost:8080/uploads/...` |

默认管理员账号：

```text
username: admin
password: password123
```

停止后端：

```bash
docker compose down
```

清空数据库、Redis 和测试数据后重新启动：

```bash

docker compose up --build -d
```

注意：`down -v` 会删除本地 Compose 数据卷，测试文章、用户、评论等都会被清空。

## 4. 请求和响应约定

所有 JSON 接口使用：

```http
Content-Type: application/json
```

图片上传使用：

```http
Content-Type: multipart/form-data
```

登录成功后，后端返回不透明 token。需要登录的接口带：

```http
Authorization: Bearer <token>
```

统一响应结构：

```json
{
  "code": "SUCCESS",
  "message": "success",
  "data": {}
}
```

分页响应结构：

```json
{
  "items": [],
  "page": 1,
  "size": 10,
  "total": 0,
  "pages": 0
}
```

前端请求封装建议：

- `code === "SUCCESS"` 时读取 `data`。
- HTTP 401 或 `code === "UNAUTHORIZED"`：清理 token，跳转登录页。
- HTTP 403 或 `code === "FORBIDDEN"`：展示无权限提示。
- `VALIDATION_ERROR`、`CONFLICT`、`UPLOAD_ERROR`：展示后端 `message`。
- 分页组件统一读取 `data.page`、`data.size`、`data.total`、`data.pages`。
- 不要展示或依赖密码明文、`passwordHash` 或数据库内部字段。
- 评论内容后端已做基础 HTML escape，前端不要把评论内容作为可信 HTML 注入。
- `contentHtml` 当前通常为 `null`，文章详情页应渲染 `contentMarkdown`。

常见错误码：

| code | HTTP | 前端处理建议 |
| --- | ---: | --- |
| `SUCCESS` | 200 | 正常读取 `data` |
| `VALIDATION_ERROR` | 400 | 表单字段错误或 toast |
| `UPLOAD_ERROR` | 400 | 展示上传失败原因 |
| `UNAUTHORIZED` | 401 | 清 token 并跳转登录 |
| `FORBIDDEN` | 403 | 展示无权限 |
| `NOT_FOUND` | 404 | 展示 404 或空状态 |
| `CONFLICT` | 409 | 展示重复/冲突提示 |
| `RATE_LIMITED` | 429 | 展示稍后再试，不要自动重试 |
| `INTERNAL_ERROR` | 500 | 展示系统异常 |

## 6. 联调测试数据准备

以下脚本会使用时间戳生成唯一数据，避免重复 slug 或用户名导致冲突。

```bash
BASE_URL=${BASE_URL:-http://localhost:8080}
SUFFIX=$(date +%s)

ADMIN_TOKEN=$(curl -s -X POST "$BASE_URL/api/auth/login" \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}' | jq -r '.data.token')

echo "ADMIN_TOKEN length: ${#ADMIN_TOKEN}"

CATEGORY_ID=$(curl -s -X POST "$BASE_URL/api/admin/categories" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H 'Content-Type: application/json' \
  -d "{\"name\":\"Tech $SUFFIX\",\"slug\":\"tech-$SUFFIX\",\"description\":\"Technical notes\",\"sortOrder\":1}" | jq -r '.data.id')

TAG_ID=$(curl -s -X POST "$BASE_URL/api/admin/tags" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H 'Content-Type: application/json' \
  -d "{\"name\":\"Spring $SUFFIX\",\"slug\":\"spring-$SUFFIX\"}" | jq -r '.data.id')

ARTICLE_SLUG="first-article-$SUFFIX"

curl -s -X POST "$BASE_URL/api/admin/articles" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H 'Content-Type: application/json' \
  -d "{
    \"title\":\"First Article $SUFFIX\",
    \"slug\":\"$ARTICLE_SLUG\",
    \"summary\":\"Article summary\",
    \"contentMarkdown\":\"# First Article\",
    \"categoryId\":$CATEGORY_ID,
    \"tagIds\":[$TAG_ID],
    \"status\":\"PUBLISHED\"
  }" | jq

GUEST_USERNAME="visitor$SUFFIX"

curl -s -X POST "$BASE_URL/api/auth/guest/register" \
  -H 'Content-Type: application/json' \
  -d "{\"username\":\"$GUEST_USERNAME\",\"password\":\"password123\",\"nickname\":\"Visitor\"}" | jq

GUEST_TOKEN=$(curl -s -X POST "$BASE_URL/api/auth/login" \
  -H 'Content-Type: application/json' \
  -d "{\"username\":\"$GUEST_USERNAME\",\"password\":\"password123\"}" | jq -r '.data.token')

curl -s -X POST "$BASE_URL/api/articles/$ARTICLE_SLUG/comments" \
  -H "Authorization: Bearer $GUEST_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"<b>Nice article!</b>"}' | jq

echo "ARTICLE_SLUG=$ARTICLE_SLUG"
echo "GUEST_USERNAME=$GUEST_USERNAME"
```

上传测试图片：

返回的 `data.url` 可用于封面、信息页或文章 Markdown。

## 7. 一键冒烟检查脚本

在启动后端后运行：

```bash
BASE_URL=${BASE_URL:-http://localhost:8080}

echo "1. cover should be SUCCESS"
curl -s "$BASE_URL/api/cover" | jq '.code'

echo "2. profile should be SUCCESS"
curl -s "$BASE_URL/api/profile" | jq '.code'

echo "3. admin endpoint without token should be UNAUTHORIZED"
curl -s "$BASE_URL/api/admin/articles" | jq '.code'

echo "4. admin login should return token"
ADMIN_TOKEN=$(curl -s -X POST "$BASE_URL/api/auth/login" \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}' | jq -r '.data.token')
test "$ADMIN_TOKEN" != "null" && test -n "$ADMIN_TOKEN" && echo "admin token ok"

echo "5. admin articles should be SUCCESS"
curl -s "$BASE_URL/api/admin/articles" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq '.code'
```

期望输出：

- 第 1、2、5 步为 `"SUCCESS"`。
- 第 3 步为 `"UNAUTHORIZED"`。
- 第 4 步输出 `admin token ok`。

## 8. 前端测试设计清单

### 8.1 公开页面

- 封面页正确展示标题、副标题、头像、背景图、链接。
- 封面链接为空或图片字段为 `null` 时页面不报错。
- 信息页正确展示名称、简介、邮箱、位置、Markdown 内容、社交链接。
- 文章列表支持默认分页、分类筛选、标签筛选、关键词搜索和空状态。
- 文章卡片在封面图为空时有降级展示。
- 文章详情正确渲染 Markdown、分类、标签、发布时间、浏览量。
- 不存在的文章 slug 展示 404 或错误页。
- 评论列表分页展示。
- 评论内容中的 HTML 按文本展示，不执行 HTML。

### 8.2 登录和权限

- 未登录访问公开页面正常。
- 未登录发表评论返回 401，前端提示登录或跳转登录页。
- 游客登录后可以发表评论。
- 游客访问 `/api/admin/**` 返回 403，前端展示无权限。
- 管理员登录后可以进入后台。
- 错误密码显示错误提示。
- 连续登录失败触发 `RATE_LIMITED` 时展示稍后再试，前端不要无限重试。
- 刷新页面后可以通过本地 token 和 `GET /api/auth/me` 恢复登录态。
- 退出登录后清除 token，受保护页面不可访问。

权限矩阵：

| 接口类型 | 未登录 | 游客 | 管理员 |
| --- | --- | --- | --- |
| 公开接口 | 200 | 200 | 200 |
| 发表评论 | 401 | 200 | 200 |
| `/api/auth/me` | 401 | 200 | 200 |
| `/api/auth/logout` | 401 | 200 | 200 |
| `/api/admin/**` | 401 | 403 | 200 |
| 上传图片 | 401 | 403 | 200 |

### 8.3 后台管理

- 管理员可创建草稿、已发布文章、隐藏文章。
- 创建已发布文章后，前台列表和详情可见。
- 草稿或隐藏文章不应出现在前台列表，前台详情返回 404。
- 编辑已发布文章不应丢失原发布时间。
- 删除文章后前台详情返回 404。
- 文章 slug 重复时展示冲突提示。
- 分类或标签不存在时展示错误提示。
- 创建分类/标签后，文章编辑页选项刷新。
- 删除仍被文章引用的分类返回 `CONFLICT`。
- 隐藏评论后，前台评论列表不再展示。
- 删除评论后，前后台列表均不展示。
- 重置游客密码后旧密码不能登录，新密码可登录。
- 禁用游客后无法登录，已有 token 失效。
- 更新封面、信息页、系统配置后，前台刷新可见。

### 8.4 图片上传

- 管理员可上传 `jpg/jpeg/png/webp/gif`。
- 未登录上传返回 401。
- 游客上传返回 403。
- 非图片文件返回 `UPLOAD_ERROR`。
- 文件扩展名和 MIME 不匹配返回 `UPLOAD_ERROR`。
- 超出大小限制返回 `UPLOAD_ERROR`。
- 上传成功后返回 `url`，浏览器可以访问图片。
- 删除图片记录后，图片列表不再展示该记录。
- 已在文章中引用的图片 URL 不会自动从 Markdown 中移除。

## 11. 常用排障

### 11.1 端口被占用

现象：容器启动失败或端口绑定失败。

检查：

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:3306 -sTCP:LISTEN
lsof -nP -iTCP:6379 -sTCP:LISTEN
```

处理：停止占用端口的进程，或修改 `docker-compose.yml` 的端口映射。

### 11.2 app 启动后立刻退出

检查日志：

```bash
docker compose logs app --tail=200
```

当前 Compose 已配置 MySQL 和 Redis healthcheck，正常情况下 app 会等待依赖 healthy 后启动。

### 11.3 测试数据冲突

现象：创建分类、标签、文章或游客时返回 `CONFLICT`。

处理：

- 换一个唯一的 `slug`、`name` 或 `username`。
- 或清空数据卷后重来：

```bash
docker compose down -v
docker compose up --build -d
```

### 11.4 前端图片无法显示

上传接口返回的 `url` 是相对路径。前端需要拼接 API origin，例如：

```text
http://localhost:8080 + /uploads/...
```

如果仍无法访问，检查文件是否上传成功：

```bash
ls -la data/uploads
```
