# 个人博客前端设计文档

版本：v0.1，待确认
依据文档：`前端开发指示.md`、`前端API接口文档.md`、`前端开发与测试交付文档.md`、`数据模型设计.md`、`API设计.md`、封面参考图 `doc/image/cover.png`

## 1. 设计目标

本前端项目用于实现一个前后端分离的个人博客网站系统，包含前台展示、游客评论登录链路、管理员后台管理链路。第一版目标是把后端已完成的主要接口完整接入，并保持页面风格接近参考封面图：现代极简、技术作品集化、浅灰背景、白色卡片、细边框、轻阴影、蓝色强调色、信息层级清晰。

后续开发前，本文件作为前端范围、页面结构、交互和接口接入的确认依据。

## 2. 技术栈

| 分类          | 选型                                                      | 用途                                    |
| ------------- | --------------------------------------------------------- | --------------------------------------- |
| 基础框架      | React + TypeScript + Vite                                 | SPA 前端工程                            |
| 路由          | React Router                                              | 前台、登录、后台路由管理                |
| 请求          | Axios                                                     | 调用 Spring Boot REST API               |
| 前台样式      | Tailwind CSS                                              | 首页、文章页、信息页等定制视觉          |
| 后台组件      | Ant Design                                                | 表格、表单、上传、Modal、分页、后台布局 |
| Markdown 编辑 | `@uiw/react-md-editor`                                  | 管理员文章编辑，左写右预览              |
| Markdown 渲染 | `react-markdown` + `remark-gfm` + `rehype-sanitize` | 前台文章与信息页渲染                    |
| 日期处理      | `dayjs`                                                 | 时间格式化                              |
| 图标          | `lucide-react` + Ant Design Icons                       | 前台按钮图标与后台常规图标              |
| 测试          | Vitest + React Testing Library + Playwright               | 单元、组件、端到端测试                  |
| 部署          | Docker + Nginx                                            | 构建静态资源并容器化部署                |

首版不引入复杂全局状态库；登录态使用 `AuthContext`，远程数据使用按模块封装的 API hooks。

## 3. 环境与请求约定

环境变量：

```text
VITE_API_BASE_URL=
```

请求封装：

- 所有接口统一经过 `src/api/http.ts` 的 Axios 实例。
- 请求自动附加 `Authorization: Bearer <token>`。
- `code === "SUCCESS"` 时返回 `data`。
- `401 / UNAUTHORIZED`：清理 token 与用户信息，跳转登录页。
- `403 / FORBIDDEN`：展示无权限提示，不自动重试。
- `VALIDATION_ERROR`、`CONFLICT`、`UPLOAD_ERROR`、`RATE_LIMITED`：展示后端 `message`。
- 上传文件使用 `multipart/form-data`。
- 本地开发默认使用 Vite `/api`、`/uploads` 同源代理，避免后端未配置 CORS 时浏览器拦截。
- 后端返回的 `/uploads/...` 相对路径通过 `toAssetUrl()` 拼接资源 origin 后展示。

登录 token 初版存储在 `localStorage`，页面刷新后通过 `GET /api/auth/me` 恢复当前用户。

| 路由                | 页面          | 权限               | 主要接口                                                            |
| ------------------- | ------------- | ------------------ | ------------------------------------------------------------------- |

| `/articles`       | 文章列表页    | PUBLIC             | `GET /api/articles`、`GET /api/categories`、`GET /api/tags`   |
| `/articles/:slug` | 文章详情页    | PUBLIC，评论需登录 | `GET /api/articles/{slug}`、评论接口                              |
| `/about`          | 信息页        | PUBLIC             | `GET /api/profile`                                                |
| `/projects`       | 项目页        | PUBLIC             | `GET /api/projects`                                                |
| `/projects/:slug` | 项目详情页    | PUBLIC             | `GET /api/projects/{slug}`                                         |
| `/login`          | 登录页        | PUBLIC             | `POST /api/auth/login`                                            |
| `/register`       | 游客注册页    | PUBLIC             | `POST /api/auth/guest/register`                                   |
| `*`               | 404 页        | PUBLIC             | 无                                                                  |

说明：项目管理已根据 `项目管理新增API文档.md` 接入真实后端接口，不再依赖前端本地缓存。

| `/admin/articles/new`      | 新建文章        | ADMIN | 分类、标签、上传、创建文章                  |
| `/admin/articles/:id/edit` | 编辑文章        | ADMIN | `GET/PUT /api/admin/articles/{id}`        |

| `/admin/comments`          | 评论管理        | ADMIN | `GET /api/admin/comments`、状态修改、删除 |
| `/admin/guests`            | 游客管理        | ADMIN | 游客列表、重置密码、修改状态                |

## 5. 前台页面设计

### 5.1 公共布局

前台使用 `PublicLayout`：

- 顶部固定导航：品牌名、Home、Blog、Projects、About、登录/后台入口。
- 移动端折叠菜单。
- 导航当前项高亮使用蓝色底线。
- 底部展示站点名、社交链接、版权信息（这些需要可以通过后台设置）。

视觉规则：

- 页面背景：浅灰 `#f5f7fb`。
- 主内容最大宽度：`1120px`。
- 卡片：白底、1px 边框、轻阴影、8px 圆角。
- 强调色：蓝色 `#2563eb`，状态色使用绿色、黄色、红色辅助，不做单一蓝色铺满。
- 字体层级克制，后台和卡片内不使用过大的标题。

### 5.2 首页 / 封面页

页面模块：

1. Hero 封面区：展示封面标题、副标题、背景图、头像或 Logo。
2. CTA：根据 `links` 生成主要入口，如阅读文章、查看项目、关于我。
3. 精选项目：首版静态展示 2-3 个项目卡片，与参考图保持作品集感。
4. 最新文章：展示最近 3 篇已发布文章。
5. 标签区：展示若干热门标签，点击进入文章列表筛选。

降级策略：

- `backgroundImageUrl` 为 `null` 时使用本地默认封面图或渐变色块加纹理感背景。
- `avatarImageUrl` 为 `null` 时用品牌首字母头像。
- `links` 为空时提供默认入口：文章、项目、信息页。

### 5.3 文章列表页

能力：

- 关键词搜索。
- 分类筛选。
- 标签筛选。
- 分页。
- 空状态、加载状态、错误重试。

文章卡片展示：

- 标题、摘要、封面图或默认占位、分类、标签、发布时间。
- 点击卡片进入详情。
- URL 查询参数保留筛选条件：`?keyword=&categorySlug=&tagSlug=&page=1`。

### 5.4 文章详情页

页面结构：

- 文章标题、摘要、封面、分类、标签、发布时间。
- Markdown 正文渲染。
- 评论区。

Markdown 安全：

- 优先渲染 `contentMarkdown`。
- `contentHtml` 当前通常为 `null`，不依赖它。
- 使用 `rehype-sanitize` 清洗。
- 文章图片中的相对 `/uploads/...` 路径自动转为完整 URL。

评论区：

- 未登录可查看评论，发表评论按钮引导登录。
- 游客和管理员登录后可评论。
- 评论内容按普通文本展示，不使用不可信 `innerHTML`。
- 评论分页默认 `size=20`。

### 5.5 信息页

页面模块：

- 头像、名称、简介、邮箱、所在地。
- 社交链接。
- `contentMarkdown` 自定义内容区。

字段为空时隐藏对应行，不展示空占位。

### 5.6 登录与注册

登录页：

- 管理员和游客共用登录表单。
- 登录成功后按角色跳转：管理员默认进入 `/admin`，游客返回来源页或首页。
- 错误密码、限流、禁用账号使用明确提示。

注册页：

- 游客注册字段：用户名、昵称、密码、确认密码。
- 注册成功后提示继续登录，不自动登录。

密码安全：

- 前端不展示、不保存、不查询任何明文密码。
- 管理员只能重置游客密码，不能查看原密码。

## 6. 后台页面设计

后台使用 Ant Design `Layout`：

- 左侧菜单：内容、互动、用户、站点、资源、系统。
- 顶部栏：当前管理员、返回前台、退出登录。
- 内容区以表格、表单、抽屉、Modal 为主，强调效率和可扫描性。

### 6.1 后台首页

展示：

- 文章总数、评论总数、图片总数、游客总数。
- 最近文章与最新评论快捷列表。
- 常用操作：新建文章、上传图片、编辑封面、编辑信息页。

统计可通过各分页接口第一页的 `total` 得出。

### 6.2 文章管理

列表：

- 筛选：状态、分类、关键词。
- 表格列：标题、状态、分类、标签、发布时间、更新时间、操作。
- 操作：编辑、删除、前台查看。

编辑页：

- 基础字段：标题、slug、摘要、封面图、分类、标签、状态。
- Markdown 编辑器：左编辑、右预览。
- 图片上传：上传成功后可一键插入 Markdown。
- 保存动作：保存草稿、发布、隐藏。

校验：

- 标题必填 1-200。
- 摘要最多 500。
- 正文必填。
- slug 最多 200，允许为空由后端生成。

### 6.3 分类与标签管理

由于后端没有单独后台列表接口，后台列表复用：

- `GET /api/categories`
- `GET /api/tags`

能力：

- 新建、编辑、删除分类。
- 新建、编辑、删除标签。
- 删除分类如返回 `CONFLICT`，提示仍有关联文章。

### 6.4 评论管理

能力：

- 筛选：状态、文章 ID、用户名。
- 展示：文章标题、评论内容、作者、IP、User-Agent、创建时间。
- 操作：设为可见、隐藏、删除。

前台隐藏评论后应从文章评论列表消失。

### 6.5 游客管理

能力：

- 搜索游客用户名。
- 查看昵称、状态、最近登录、创建时间。
- 重置密码。
- 启用 / 禁用游客。

禁用游客或重置密码后，该游客已有 token 失效。前端只提示操作结果，不展示密码哈希。

### 6.6 封面配置

表单字段：

- 标题、副标题。
- 背景图 URL、头像 URL。
- 链接列表：label、url、type、sortOrder。

图片上传：

- 背景图使用 `usageType=COVER`。
- 头像建议使用 `usageType=PROFILE`。
- 上传后将返回 URL 自动填入对应字段。

### 6.7 信息页配置

表单字段：

- displayName、bio、avatarImageUrl、email、location。
- socialLinks。
- contentMarkdown，带预览。

### 6.8 图片资源管理

能力：

- 按 `usageType` 筛选。
- 上传图片。
- 复制 URL。
- 预览图片。
- 删除图片记录。

注意：删除图片只软删除记录，不会自动修改已引用图片的文章或配置。

### 6.9 系统配置

展示并编辑已有系统配置：

- `site.title`
- `site.description`
- `comment.defaultStatus`
- `upload.maxFileSizeMb`
- `upload.allowedImageTypes`

不支持新增或修改 `configKey`。

## 9. 权限与状态流

### 9.1 用户状态

| 状态   | 前台访问     | 评论     | 后台       |
| ------ | ------------ | -------- | ---------- |
| 未登录 | 可访问公开页 | 引导登录 | 跳转登录   |
| GUEST  | 可访问公开页 | 可评论   | 无权限     |
| ADMIN  | 可访问公开页 | 可评论   | 可进入后台 |

### 9.2 登录恢复

1. 应用启动读取本地 token。
2. 如果存在 token，调用 `GET /api/auth/me`。
3. 成功则设置当前用户。
4. 失败则清理 token。

### 9.3 退出登录

1. 调用 `POST /api/auth/logout`。
2. 无论接口成功或 token 已失效，都清理本地登录态。
3. 返回首页或登录页。

## 10. 错误、加载与空状态

公开页面：

- 加载时使用骨架屏。
- 列表为空时展示空状态与清除筛选按钮。
- 404 文章详情显示“文章不存在或尚未发布”。
- 网络失败提供重试按钮。

后台页面：

- 表格使用 Ant Design loading。
- 表单保存时禁用提交按钮。
- 删除动作使用二次确认。
- 保存成功 toast，失败展示后端 `message`。

## 12. Docker 部署设计

前端 Docker 镜像建议：

1. Node 阶段执行依赖安装与 `npm run build`。
2. Nginx 阶段托管 `dist`。
3. Nginx 配置 SPA fallback 到 `index.html`。
4. API 地址通过构建环境变量或运行时配置注入。

本地开发使用 Vite dev server，生产部署使用 Nginx 静态服务。
