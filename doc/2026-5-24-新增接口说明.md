# 2026-5-24 新增接口说明

## 1. 接口用途

本次新增后台管理员资料接口，用于修改当前登录管理员的昵称。前端可以在后台设置页、账户菜单或个人资料弹窗中增加“昵称”输入项，提交后用接口返回的用户信息更新当前登录用户状态。

该接口只修改管理员昵称，不修改登录用户名、密码或头像。

## 2. 接口信息

| 项目 | 内容 |
| --- | --- |
| Method | `PATCH` |
| URL | `/api/admin/me/profile` |
| 权限 | 管理员登录态 |
| Content-Type | `application/json` |
| Authorization | `Bearer <admin-token>` |

### 请求体

```json
{
  "nickname": "Sam"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `nickname` | string | 是 | 管理员展示昵称，长度 1-64 |

### 成功响应

```json
{
  "code": "SUCCESS",
  "message": "success",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "Sam",
    "role": "ADMIN"
  }
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `data.id` | 当前管理员用户 ID |
| `data.username` | 管理员登录用户名，不会被本接口修改 |
| `data.nickname` | 更新后的管理员昵称 |
| `data.role` | 当前用户角色，管理员为 `ADMIN` |

## 3. 前端修改建议

1. 在后台设置页或右上角账户菜单中增加“管理员昵称”设置入口。
2. 页面初始化时可以继续使用现有 `/api/auth/me` 获取当前用户信息，将 `nickname` 回填到输入框。
3. 用户提交时调用 `PATCH /api/admin/me/profile`，请求体只传 `nickname`。
4. 接口返回 `SUCCESS` 后，直接使用 `data` 更新前端当前用户状态和页面展示昵称。
5. 本接口不会强制管理员重新登录。若前端后续再次调用 `/api/auth/me`，可能仍取到 token 中的旧昵称；建议在本次修改成功后以前端本地用户状态中的新昵称为准。
6. 管理员昵称修改后，后续评论列表中的 `author.nickname` 会展示新昵称。

## 4. 错误处理

| 场景 | HTTP 状态 | 说明 |
| --- | --- | --- |
| 未登录 | `401` | 缺少或无效 token |
| 非管理员账号 | `403` | 游客账号无权访问 `/api/admin/**` |
| 昵称为空 | `400` | `nickname` 必填 |
| 昵称超过 64 字符 | `400` | `nickname` 长度限制为 1-64 |

前端建议：

- `401`：跳转登录页或提示登录过期。
- `403`：提示无权限。
- `400`：在表单中展示后端返回的校验错误信息。

## 5. 联调示例

### 管理员登录

```bash
BASE_URL=${BASE_URL:-http://localhost:8080}

ADMIN_TOKEN=$(curl -s -X POST "$BASE_URL/api/auth/login" \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}' | jq -r '.data.token')
```

### 修改管理员昵称

```bash
curl -s -X PATCH "$BASE_URL/api/admin/me/profile" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"nickname":"Sam"}' | jq
```

期望返回：

```json
{
  "code": "SUCCESS",
  "message": "success",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "Sam",
    "role": "ADMIN"
  }
}
```

## 6. 验收步骤

1. 管理员登录后台。
2. 打开前端新增的管理员昵称设置入口。
3. 输入新昵称并保存。
4. 页面上的当前用户昵称立即更新为新值。
5. 使用管理员账号在文章详情页发表评论。
6. 获取文章评论列表，确认该评论的 `author.nickname` 为新昵称。
7. 确认管理员仍可使用原 `username` 登录，登录用户名未被修改。
