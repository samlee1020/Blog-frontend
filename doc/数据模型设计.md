# 个人博客网站系统数据模型设计

## 1. 设计目标

本文档描述个人博客网站系统的核心数据模型，覆盖封面页、文章、信息页、登录、游客评论、后台管理和图片上传等功能。

后端技术栈按 Spring Boot + MySQL + Redis 设计，部署方式按 Docker 预留。前端技术栈未定，因此数据模型保持通用 REST API 友好。

重要约定：

- 用户分为管理员 `ADMIN` 和游客 `GUEST`。
- 游客仅用于登录后发表评论，不设计游客个人主页等额外功能。
- 密码不明文存储，数据库只保存密码哈希。前台评论只展示用户名或昵称，不展示密码。
- 删除类操作默认采用软删除，便于后台恢复和审计。
- 图片先按服务器本地或 Docker 挂载目录存储，通过 URL 访问；后续可替换为对象存储。
- 数据库字段统一使用 `snake_case`，API JSON 字段统一使用 `camelCase`。

## 2. 核心实体关系

### 2.1 实体概览

| 实体               | 说明                                   |
| ------------------ | -------------------------------------- |
| `user`           | 系统用户，包括管理员和游客             |
| `article`        | 博客文章，保存 Markdown 内容和发布状态 |
| `category`       | 文章分类                               |
| `tag`            | 文章标签                               |
| `article_tag`    | 文章和标签的多对多关系                 |
| `comment`        | 游客对文章的评论                       |
| `cover_config`   | 封面页配置，包括背景图、标题、链接     |
| `profile_config` | 信息页配置，展示个人信息               |
| `media_asset`    | 图片等上传资源                         |
| `system_config`  | 通用系统配置                           |

### 2.2 关系说明

- 一个用户可以发表多条评论。
- 一篇文章可以拥有多条评论。
- 一篇文章可以属于一个分类。
- 一篇文章可以绑定多个标签。
- 上传图片会生成一条 `media_asset` 记录，文章 Markdown、封面配置、信息页配置可通过图片 URL 引用资源。
- 封面页和信息页按配置表设计，便于后台修改内容而不需要重新部署。

## 3. MySQL 表结构

### 3.1 用户表 `user`

保存管理员和游客账号。游客账号只用于评论登录。

| 字段              | 类型         | 约束               | 说明                      |
| ----------------- | ------------ | ------------------ | ------------------------- |
| `id`            | BIGINT       | PK, AUTO_INCREMENT | 用户 ID                   |
| `username`      | VARCHAR(64)  | NOT NULL, UNIQUE   | 登录用户名                |
| `nickname`      | VARCHAR(64)  | NOT NULL           | 展示名称                  |
| `password_hash` | VARCHAR(255) | NOT NULL           | 密码哈希                  |
| `role`          | VARCHAR(16)  | NOT NULL           | `ADMIN` / `GUEST`     |
| `status`        | VARCHAR(16)  | NOT NULL           | `ACTIVE` / `DISABLED` |
| `last_login_at` | DATETIME     | NULL               | 最近登录时间              |
| `created_at`    | DATETIME     | NOT NULL           | 创建时间                  |
| `updated_at`    | DATETIME     | NOT NULL           | 更新时间                  |
| `deleted_at`    | DATETIME     | NULL               | 软删除时间                |

索引：

- 唯一索引：`uk_user_username(username)`
- 普通索引：`idx_user_role(role)`
- 普通索引：`idx_user_status(status)`

### 3.2 文章表 `article`

保存文章基础信息和 Markdown 正文。

| 字段                 | 类型         | 约束               | 说明                                   |
| -------------------- | ------------ | ------------------ | -------------------------------------- |
| `id`               | BIGINT       | PK, AUTO_INCREMENT | 文章 ID                                |
| `title`            | VARCHAR(200) | NOT NULL           | 标题                                   |
| `slug`             | VARCHAR(200) | NOT NULL, UNIQUE   | URL 友好的唯一标识                     |
| `summary`          | VARCHAR(500) | NULL               | 摘要                                   |
| `cover_image_url`  | VARCHAR(500) | NULL               | 文章封面图 URL                         |
| `content_markdown` | LONGTEXT     | NOT NULL           | Markdown 正文                          |
| `content_html`     | LONGTEXT     | NULL               | 渲染后的 HTML 缓存，第一版可为空       |
| `category_id`      | BIGINT       | NULL               | 分类 ID                                |
| `status`           | VARCHAR(16)  | NOT NULL           | `DRAFT` / `PUBLISHED` / `HIDDEN`       |
| `view_count`       | BIGINT       | NOT NULL DEFAULT 0 | 浏览量                                 |
| `published_at`     | DATETIME     | NULL               | 发布时间                               |
| `created_by`       | BIGINT       | NOT NULL           | 创建管理员 ID                          |
| `updated_by`       | BIGINT       | NULL               | 最近更新管理员 ID                      |
| `created_at`       | DATETIME     | NOT NULL           | 创建时间                               |
| `updated_at`       | DATETIME     | NOT NULL           | 更新时间                               |
| `deleted_at`       | DATETIME     | NULL               | 软删除时间                             |

索引：

- 唯一索引：`uk_article_slug(slug)`
- 普通索引：`idx_article_status_published_at(status, published_at)`
- 普通索引：`idx_article_category_id(category_id)`
- 普通索引：`idx_article_created_by(created_by)`

说明：

- 前台只展示 `status = PUBLISHED` 且未删除的文章。
- 管理后台可查看草稿、已发布、隐藏文章。
- `content_html` 可不在第一版实现。如果由前端实时渲染 Markdown，可保持为空。

### 3.3 分类表 `category`

文章分类，一篇文章最多属于一个分类。

| 字段            | 类型         | 约束               | 说明               |
| --------------- | ------------ | ------------------ | ------------------ |
| `id`          | BIGINT       | PK, AUTO_INCREMENT | 分类 ID            |
| `name`        | VARCHAR(64)  | NOT NULL           | 分类名称           |
| `slug`        | VARCHAR(100) | NOT NULL, UNIQUE   | URL 标识           |
| `description` | VARCHAR(255) | NULL               | 描述               |
| `sort_order`  | INT          | NOT NULL DEFAULT 0 | 排序值，越小越靠前 |
| `created_at`  | DATETIME     | NOT NULL           | 创建时间           |
| `updated_at`  | DATETIME     | NOT NULL           | 更新时间           |
| `deleted_at`  | DATETIME     | NULL               | 软删除时间         |

索引：

- 唯一索引：`uk_category_slug(slug)`
- 普通索引：`idx_category_sort_order(sort_order)`

### 3.4 标签表 `tag`

文章标签，和文章是多对多关系。

| 字段           | 类型         | 约束               | 说明       |
| -------------- | ------------ | ------------------ | ---------- |
| `id`         | BIGINT       | PK, AUTO_INCREMENT | 标签 ID    |
| `name`       | VARCHAR(64)  | NOT NULL, UNIQUE   | 标签名称   |
| `slug`       | VARCHAR(100) | NOT NULL, UNIQUE   | URL 标识   |
| `created_at` | DATETIME     | NOT NULL           | 创建时间   |
| `updated_at` | DATETIME     | NOT NULL           | 更新时间   |
| `deleted_at` | DATETIME     | NULL               | 软删除时间 |

索引：

- 唯一索引：`uk_tag_name(name)`
- 唯一索引：`uk_tag_slug(slug)`

### 3.5 文章标签关系表 `article_tag`

| 字段           | 类型     | 约束               | 说明     |
| -------------- | -------- | ------------------ | -------- |
| `id`         | BIGINT   | PK, AUTO_INCREMENT | 关系 ID  |
| `article_id` | BIGINT   | NOT NULL           | 文章 ID  |
| `tag_id`     | BIGINT   | NOT NULL           | 标签 ID  |
| `created_at` | DATETIME | NOT NULL           | 创建时间 |

索引：

- 唯一索引：`uk_article_tag(article_id, tag_id)`
- 普通索引：`idx_article_tag_tag_id(tag_id)`

### 3.6 评论表 `comment`

游客登录后可对文章发表评论。

| 字段           | 类型          | 约束               | 说明                                   |
| -------------- | ------------- | ------------------ | -------------------------------------- |
| `id`         | BIGINT        | PK, AUTO_INCREMENT | 评论 ID                                |
| `article_id` | BIGINT        | NOT NULL           | 文章 ID                                |
| `user_id`    | BIGINT        | NOT NULL           | 评论用户 ID                            |
| `content`    | VARCHAR(2000) | NOT NULL           | 评论内容                               |
| `status`     | VARCHAR(16)   | NOT NULL           | `VISIBLE` / `HIDDEN` / `PENDING`       |
| `ip_address` | VARCHAR(64)   | NULL               | 评论 IP，后台审计使用                  |
| `user_agent` | VARCHAR(500)  | NULL               | 浏览器 UA，后台审计使用                |
| `created_at` | DATETIME      | NOT NULL           | 创建时间                               |
| `updated_at` | DATETIME      | NOT NULL           | 更新时间                               |
| `deleted_at` | DATETIME      | NULL               | 软删除时间                             |

索引：

- 普通索引：`idx_comment_article_status_created(article_id, status, created_at)`
- 普通索引：`idx_comment_user_id(user_id)`
- 普通索引：`idx_comment_status(status)`

说明：

- 第一版评论默认使用 `VISIBLE`，直接展示。
- 如果后续需要审核，可把新评论默认状态调整为 `PENDING`。
- 前台评论列表返回用户 `nickname` 或 `username`，不返回 `password_hash`。

### 3.7 封面配置表 `cover_config`

保存封面页展示内容。系统通常只有一条启用配置。

| 字段                     | 类型         | 约束               | 说明            |
| ------------------------ | ------------ | ------------------ | --------------- |

| `avatar_image_url`     | VARCHAR(500) | NULL               | 头像或 Logo URL |
| `links_json`           | JSON         | NULL               | 封面链接列表    |
| `is_active`            | TINYINT(1)   | NOT NULL DEFAULT 1 | 是否启用        |
| `created_at`           | DATETIME     | NOT NULL           | 创建时间        |
| `updated_at`           | DATETIME     | NOT NULL           | 更新时间        |

`links_json` 示例：

```json
[
  {
    "label": "Articles",
    "url": "/articles",
    "type": "internal",

  },
  {
    "label": "GitHub",
    "url": "https://github.com/example",
    "type": "external",
```

索引：

- 普通索引：`idx_cover_config_active(is_active)`

## 4. 文件存储约定

第一版图片上传使用本地文件系统，部署时通过 Docker volume 挂载持久化目录。

| 配置项 | 建议值 | 说明 |
| --- | --- | --- |
| 容器内上传根目录 | `/app/uploads` | Spring Boot 保存文件的位置 |
| 宿主机挂载目录 | `./data/uploads:/app/uploads` | Docker Compose volume 建议配置 |
| 静态访问前缀 | `/uploads/**` | Web 访问上传文件的 URL 前缀 |
| 文章图片目录 | `/app/uploads/article/yyyy/MM/dd` | `usage_type = ARTICLE` |
| 封面图片目录 | `/app/uploads/cover/yyyy/MM/dd` | `usage_type = COVER` |
| 信息页图片目录 | `/app/uploads/profile/yyyy/MM/dd` | `usage_type = PROFILE` |

`media_asset.storage_path` 保存容器内真实路径，`media_asset.url` 保存前端可访问 URL，例如 `/uploads/article/2026/05/04/uuid.png`。

## 6. Redis 设计

### 6.1 登录 Token

用于保存登录态，配合 HTTP Header `Authorization: Bearer <token>` 使用。

| Key                    | Value         | TTL  | 说明                      |
| ---------------------- | ------------- | ---- | ------------------------- |
| `auth:token:{token}` | 用户会话 JSON | 7 天 | 登录 token 对应的用户信息 |

Value 示例：

```json
{
  "userId": 1,
  "username": "admin",
  "nickname": "Admin",
  "role": "ADMIN",
  "loginAt": "2026-05-04T10:00:00"
}
```

### 6.2 登录限流

用于减少暴力破解风险。

| Key                            | Value    | TTL     | 说明         |
| ------------------------------ | -------- | ------- | ------------ |
| `rate:login:{username}:{ip}` | 失败次数 | 15 分钟 | 登录失败计数 |

建议策略：

- 同一用户名和 IP 在 15 分钟内失败 5 次后暂时拒绝登录。
- 登录成功后删除对应失败计数。

### 6.3 文章缓存

| Key                                         | Value         | TTL     | 说明             |
| ------------------------------------------- | ------------- | ------- | ---------------- |
| `article:detail:{slug}`                   | 文章详情 JSON | 10 分钟 | 前台文章详情缓存 |
| `article:list:{page}:{size}:{filterHash}` | 文章列表 JSON | 5 分钟  | 前台文章列表缓存 |

文章发布、编辑、删除后需要删除相关缓存。

### 6.4 站点配置缓存

| Key              | Value               | TTL     | 说明         |
| ---------------- | ------------------- | ------- | ------------ |
| `site:cover`   | 当前封面配置 JSON   | 30 分钟 | 封面页缓存   |
| `site:profile` | 当前信息页配置 JSON | 30 分钟 | 信息页缓存   |
| `site:config`  | 通用站点配置 JSON   | 30 分钟 | 系统配置缓存 |

后台修改配置后需要删除对应缓存。

## 7. 数据安全和一致性

- 密码使用 BCrypt 或 Argon2 哈希存储。
- 接口响应永远不返回 `password_hash`。
- 管理员可重置游客密码，也可修改自己的管理员密码，但不可查看任何用户的原始密码。
- 删除文章时不物理删除评论；文章软删除后前台不再展示文章和评论。
- 删除图片资源时建议先软删除数据库记录，再删除物理文件；如果物理删除失败，需要记录错误日志。
- 更新文章标签时先删除旧关系，再写入新关系，建议放在同一个事务内。
- 发布文章时如果 `slug` 为空，可根据标题生成，并处理重复冲突。

## 8. 初始化数据

系统首次启动需要准备：

- 一个管理员账号，来源可以是初始化 SQL、环境变量或启动脚本。
- 一条默认封面配置。
- 一条默认信息页配置。
- 基础系统配置项，例如站点标题和上传限制。
